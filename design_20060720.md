## 4. Server 层设计

### 4.1 新增 Service

**`src/server/services/ai.service.ts`**

```typescript
import 'server-only';
import * as configService from './site-config.service';

const PROMPT_TEMPLATE = `Task: Extract the summary and keywords from the article below.

Input:
{
  "article": "CONTENT_PLACEHOLDER"
}

Output Requirements:
- Return ONLY a JSON object containing summary and keywords.
- Summary: A single sentence of fewer than 50 characters.
- Keywords: 3–5 keywords.
- Output Language: Chinese.
- Content: The response should begin with { and end with }.
- Do not include any introductory text, markdown explanations, or notes.
- No Markdown: Do not wrap the JSON in markdown formatting.
- Strict Output Schema:
    {
      "summary": "本文是关于数组解构、合并展开及reduce的统计",
      "keywords": ["Name 1", "Name 2"]
    }`;

export async function extractSummary(content: string): Promise<{ summary: string; keywords: string[] }> {
  const url = await configService.getSiteConfig('ai_provider_url');
  const apiKey = await configService.getSiteConfig('ai_api_key');
  const model = (await configService.getSiteConfig('ai_model')) || 'gpt-4o-mini';

  if (!url || !apiKey) throw new Error('AI Provider 未配置');

  const truncated = content.length > 8000 ? content.slice(0, 8000) : content;
  const prompt = PROMPT_TEMPLATE.replace('CONTENT_PLACEHOLDER', truncated);

  const response = await fetch(url, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${apiKey}`,
    },
    body: JSON.stringify({
      model,
      messages: [
        { role: 'user', content: prompt },
      ],
      temperature: 0.3,
      max_tokens: 200,
    }),
    signal: AbortSignal.timeout(30000),
  });

  if (!response.ok) {
    throw new Error(`AI API 返回错误: ${response.status}`);
  }

  const data = await response.json();
  const text = data.choices?.[0]?.message?.content || '';

  // 解析 JSON — 兼容可能被 markdown 包裹的情况
  const jsonMatch = text.match(/\{[\s\S]*\}/);
  if (!jsonMatch) throw new Error('AI 返回格式异常');
  const result = JSON.parse(jsonMatch[0]);

  return {
    summary: result.summary || '',
    keywords: Array.isArray(result.keywords) ? result.keywords.slice(0, 5) : [],
  };
}
```

import bcrypt from 'bcryptjs';

export async function hashPassword(password: string): Promise<string> {
  return bcrypt.hash(password, 12);
}

export async function verifyPassword(password: string, hash: string): Promise<boolean> {
  return bcrypt.compare(password, hash);
}

### 4.4a 分享链接 — Repository 新增方法

**`src/server/repositories/post.repository.ts`**

```typescript
export async function findByShareToken(token: string) {
  const supabase = createAdminClient();
  const { data } = await supabase
    .from('posts')
    .select('*')
    .eq('share_token', token)
    .single();
  return data;
}

export async function regenerateShareToken(postId: string): Promise<string> {
  const supabase = createAdminClient();
  const newToken = crypto.randomUUID();
  await supabase.from('posts').update({ share_token: newToken }).eq('id', postId);
  return newToken;
}
```

### 4.4b 分享链接 — 服务端路由验证

**`src/app/(public)/blog/[slug]/page.tsx`** — token 验证逻辑：

```
访问 /blog/[slug]
  → ?token=xxx 存在 → findByShareToken(token) → 匹配 → 直接渲染文章内容
  → token 不匹配 → 渲染 PasswordGate
  → 无 token → 现有逻辑 (public → 渲染, private → 404, password → PasswordGate)
```

分享链接跳过密码验证，直接返回文章完整内容。不设置 cookie（token 在 URL 中已足够）。

### 4.5 API Routes

#### `src/app/api/mcp/route.ts`

处理 MCP JSON-RPC 请求，支持以下方法：
- `initialize` — 返回 server info + capabilities
- `tools/list` — 返回可用 tools 列表
- `tools/call` — 调用具体 tool

实现要点：
- 提取 `Authorization: Bearer <token>`，比对 `site_config.mcp_api_key`
- 解析 JSON-RPC body，分发到对应 handler
- 返回 SSE 流式响应（`Content-Type: text/event-stream`）

#### `src/app/api/verify-password/route.ts`

```
POST /api/verify-password
Body: { postId: string, password: string }

验证流程:
1. 查询 post (by id, admin client)
2. 如果 visibility !== 'password' → 400
3. bcrypt.compare(password, post.password_hash)
4. 匹配成功 → 设置 cookie post_read_<postId>=<token> (httpOnly, maxAge=3600)
5. 返回 { success: true, post: { ... } } 或 { success: false, error: '密码错误' }
```

## 6. MCP Server 详细设计

### 6.1 端点处理流程

```
POST /api/mcp
├── 1. 验证 Authorization header
│   └── 无效 → 401 { error: "Unauthorized" }
├── 2. 解析 JSON-RPC body
│   └── 无效 → 400 { error: "Invalid JSON-RPC" }
├── 3. 路由到 method handler
│   ├── initialize → MCP server info
│   ├── tools/list → 返回 tools schema 数组
│   ├── tools/call → 调用具体 tool
│   └── 未知 → 404
└── 4. 返回 SSE 流
    └── Content-Type: text/event-stream
```

### 6.2 MCP Tools 实现映射

| Tool Name | 调用现有 Service | 额外处理 |
|-----------|------------------|---------|
| `create_draft` | `postService.savePost()` | 自动设置 status=draft, author=service_role user |
| `publish_post` | `postService.savePost()` | 传入 postId + 新数据 |
| `list_posts` | `postService.listPosts()` | 支持 status/visibility/limit/offset 过滤 |
| `get_post` | `postService.getPost()` | 返回 PostWithRelations |
| `delete_post` | `postService.deletePost()` | 验证 post 存在 |
| `search_posts` | `searchService.searchPosts()` | 全文搜索 |
| `get_stats` | `statsService.getDashboardStats()` | 汇总统计 |

### 6.3 认证与权限

```typescript
// middleware/auth-mcp.ts
export async function authenticateMcp(request: NextRequest): Promise<boolean> {
  const authHeader = request.headers.get('authorization');
  if (!authHeader?.startsWith('Bearer ')) return false;
  const token = authHeader.slice(7);
  const storedKey = await configService.getSiteConfig('mcp_api_key');
  return token === storedKey && storedKey !== '';
}
```
