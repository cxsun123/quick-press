## 📊 各层依赖关系（由上到下）

```
┌─────────────────────────────────────────────────┐
│              app/ (Pages & API Routes)          │
│   - Server Components 调用 Service              │
│   - API Routes 调用 Service                     │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│              server/services/                    │
│   - 业务逻辑、事务、权限检查                     │
│   - 调用 Repository + Guard                     │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│            server/repositories/                  │
│   - 纯 CRUD 操作                                │
│   - 调用 Database Client                        │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│               server/db/                        │
│   - 数据库连接、Schema                          │
└─────────────────────────────────────────────────┘
```

## 💡 结构优势总结

1. **关注点分离**：Repository → Service → API/Page 三层清晰
2. **类型安全**：模型全栈共享，Zod 校验统一
3. **安全隔离**：`server-only` 保护后端代码
4. **可扩展性**：新功能只需添加对应模块
5. **测试友好**：每层都可独立测试
6. **认证完整**：从中间件到守卫全覆盖
