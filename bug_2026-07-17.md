# Bug 2026-07-17

## Bug 1: 嵌入式编辑器工具栏样式丢失

### 状态: FIXED

### 描述
i_blog 后台编辑器（`/admin/posts/new`）的工具栏样式异常：
- 按钮间距为 0，所有按钮挤在一起
- 工具栏背景色丢失
- 颜色指示器（黄色）被拉伸成 160px×10px 的大色块

### 复现
1. 登录 `/login`（chengxin.sun@gmail.com / admin123）
2. 访问 `/admin/posts/new`
3. 工具栏按钮无间距、背景透明、背景色指示器为大黄块

### 期望行为
工具栏应有 8px 内边距、4px 按钮间距、灰色背景（#f9fafb）、颜色指示器为 12px×4px 小条。

### 实际行为
工具栏无背景、按钮挤在一起、颜色指示器 160px×10px。

### 根因
Tailwind CSS v4 默认不扫描 `node_modules`，编辑器组件 `@chengxinsun26/editor`（源码以 .tsx 直接发布在 node_modules 中）使用的工具类（`bg-gray-50`、`w-3`、`h-1`、`shadow-lg` 等）未生成。

最初尝试用 globals.css 结构化选择器手动补样式，但给 `.absolute` 加了 `min-width: 160px`，导致颜色指示器被拉伸。

### 修复方案
在 `src/app/globals.css` 加一行 `@source`，让 Tailwind v4 扫描编辑器包源码并自动生成所有工具类：

```css
@source "../../node_modules/@chengxinsun26/editor/src";
```

删除所有手动补丁的 CSS。

### 涉及文件
- `src/app/globals.css`

### 验证结果
- 工具栏 28 个子元素正常排列，无超大元素
- 按钮 28×28px，间距 4px
- 颜色指示器恢复为 12px×4px
- CSS 体积从 49KB 增至 64KB（工具类已生成）

## Bug 2: 文章页链接无链接样式

### 状态: FIXED

### 描述
文章页（`/blog/test1`）中所有 `<a>` 链接与正文颜色一致（rgb(23,23,23)），无下划线、无蓝色，无法分辨。

### 复现
1. 访问 `http://localhost:3000/blog/test1`
2. 文中"🌐 产品介绍"等链接为正文样式

### 期望行为
链接应显示为蓝色（--primary = #3b82f6）、带下划线、字重 500。

### 实际行为
链接颜色 rgb(23,23,23)（前景色）、无下划线、字重 400。

### 根因
i_blog 未安装 `@tailwindcss/typography` 插件，`.prose` 及 `prose-a:` 等变体完全没有样式，page.tsx 里写的 `prose-a:text-[var(--primary)]` 等不生效。

### 修复方案
1. 安装插件：`pnpm add @tailwindcss/typography`
2. 在 `src/app/globals.css` 加 `@plugin "@tailwindcss/typography";`（Tailwind v4 用 `@plugin` 而非 PostCSS 配置）
3. 删除失效的 `prose-a:no-underline hover:prose-a:underline`（接受 typography 默认带下划线的链接样式）

### 涉及文件
- `package.json`
- `src/app/globals.css`
- `src/app/(public)/blog/[slug]/page.tsx`

### 验证结果
8 个链接样本全部 `color: rgb(59,130,246)`、`text-decoration: underline`、`font-weight: 500` ✓

## Bug 3: 文章页表格无边框

### 状态: FIXED

### 描述
文章页表格无可见边框线，单元格 padding 不对称（右侧 0px），布局拥挤。

### 复现
1. 访问 `http://localhost:3000/blog/test1`
2. 表格单元格 border 为 0px

### 期望行为
表格应有边框，单元格对称 padding。

### 实际行为
table/td/th 的 border 全为 0px，仅 thead 有底边框。

### 根因
typography 插件的默认表格边框在 Tailwind v4 下未生效，page.tsx 写的 `prose-table:border-[var(--border)]` 等覆盖也没生效。

### 修复方案
1. 删除 page.tsx 中失效的 `prose-table:`、`prose-th:`、`prose-td:` 变体（保留能生效的 `prose-th:bg-[var(--muted)]` 移到 globals.css）
2. 在 globals.css 用显式 CSS 给 `.prose table/th/td` 加边框和对称 padding：

```css
.prose table {
  width: 100%;
  border-collapse: collapse;
  border: 1px solid var(--border);
}
.prose th, .prose td {
  border: 1px solid var(--border);
  padding: 8px 12px;
}
.prose th {
  background-color: var(--muted);
  font-weight: 600;
}
```

### 涉及文件
- `src/app/globals.css`
- `src/app/(public)/blog/[slug]/page.tsx`

### 验证结果
- table 四边 1px solid border ✓
- td border 1px solid、padding 8px 12px（对称）✓
- th border 1px solid、padding 8px 12px、bg var(--muted)、fontWeight 600 ✓
- 14 张表 / 145 个单元格全部带边框 ✓
